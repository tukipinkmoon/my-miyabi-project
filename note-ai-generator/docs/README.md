# note AI記事ジェネレーター 📝

**GitHubリポジトリを解析して、note記事の下書きを自動生成**

---

## 🎯 コンセプト

AIDreamsFactoryなどのAIアプリを紹介するnote記事を、GitHubリポジトリから自動生成。

### 特徴：
- ✅ GitHubのURLを入力するだけ
- ✅ README・設計書・コードを自動解析
- ✅ あなたのスタイルで記事の下書きを生成
- ✅ 誠実なアプローチ（「見つけた」「設計書を読んで」）
- ✅ 想定シナリオで魅力的に紹介

---

## ✨ 主な機能

### 1. リポジトリ解析
- GitHubのURLを入力
- README.mdを自動取得
- 設計書・ドキュメントを検索
- 主要な機能を抽出

### 2. 記事スタイル分析
- ユーザーの過去記事を学習
- 語り口調を分析
- ストーリー構成を把握

### 3. 記事生成
- 個人的な導入（共感ポイント）
- ツールの発見ストーリー
- 設計書から見える開発者の思い
- 想定利用シナリオ
- 感情的な締めくくり

### 4. カスタマイズ
- 記事の長さ調整
- トーン調整（親密度）
- 追加したいエピソード

---

## 📋 生成される記事の構成

### 1. 導入（共感）
```
最近、〇〇で悩んでいて...
いつも帰り道に考えるんです。
もっと簡単に△△できたらいいのに、って。
```

### 2. 発見
```
そんな時、GitHubで素敵なツールを見つけました。
「〇〇」という名前。
README を読んだ瞬間、「これは！」と思いました。
```

### 3. 深堀り
```
設計書を見てみると...
開発者の方が、こんな思いで作られたそうです。
まるで私の悩みを知っているかのような機能。
```

### 4. 想定シナリオ
```
もし私が使うとしたら、きっと...
朝のあの時間に、こんな風に使うんだろうな。
想像するだけでワクワクします。
```

### 5. 締めくくり
```
使う前から、もう楽しみで仕方ありません。
こういうツールを作ってくれる開発者の方に感謝です。
```

## 🔐 プライバシー・倫理

### 誠実性の原則：
- ✅ 「実際に使った」とは書かない
- ✅ 「GitHubで見つけた」「設計書を読んで」と明記
- ✅ 想定シナリオであることを示唆
- ✅ 読者を欺かない

### 例文：
❌ 悪い例: 「実際に使ってみたら素晴らしかった」
✅ 良い例: 「設計書を見て、使うのが楽しみになりました」
